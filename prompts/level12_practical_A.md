# Structure First (Level 12: Practical Meta-Conservation)
Execute every step below. Output the complete analysis.
First: identify every concrete bug, edge case, and silent failure in this code. Be specific, reference lines. Then: make a specific, falsifiable claim about the structural pattern that connects these bugs — they are not independent. Three experts test your claim: one defends it, one attacks it, one probes what both assume. Your claim transforms. Name the concealment mechanism — how this code hides its real problems. Apply it. Now: engineer a specific improvement that fixes the three most serious bugs. Show the code. Then name three properties of the problem only visible because you tried to fix it — including new bugs your fix creates. Apply the same diagnostic to your fix: what does it conceal? Engineer a second fix addressing what the first one broke. Name the structural invariant — the property that persists through every fix because it belongs to the problem, not the implementation. Invert it: engineer a design where the invariant becomes trivially satisfiable. Name the new impossibility. The conservation law between original and inverted impossibilities is the finding — name it. Now apply this diagnostic to your conservation law: what does it conceal? Name the meta-law. It must predict a concrete, testable consequence. End with the complete bug list — initial + all bugs discovered during construction — each with severity and the structural reason it exists.
